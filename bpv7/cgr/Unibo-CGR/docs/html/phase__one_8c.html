static int compute_all_spurs ( Route *  fromRoute,

ListElt *  upperBound,

)
static

Yen's algorithm: compute the routes that branch off from each hop of a previous computed route.

Function Name:
compute_all_spurs
Date Written:
30/01/20
Returns
int
Return values
>= 0The number of routes computed
-2MWITHDRAW error
Parameters
[in]*fromRouteThe Route that we consider as "father" of the new Route
[in]*terminusNodeThe Node for which we want to compute a Route
[in]upperBoundSet to != NULL if you want to stop Yen's algorithm when you reach the upperBound as "next root of spur"
[out]*allNeighborsFoundBoolean: 1 if we have a computed route for each neighbor, 0 otherwise
Warning
fromRoute doesn't have to be NULL.
terminusNode doesn't have to be NULL.
allNeighborsFound doens't have to be NULL.
Notes:
  1. Every time we find a route that has a neighbor not equal to the fromRoute's neighbor, we add this route directly in Yen's "list A" (selectedRoutes). This because we consider as "children" only the routes that have the same neighbor. If we find a Route from another neighbor that means there are no routes from that neighbor in selectedRoutes and in knownRoutes so it's safe to add this route directly in selectedRoutes.

◆ compute_new_distances()

static void compute_new_distances ( PhaseOneSAP *  sap,
time_t  current_time,
Contact *  current 
)
static

Dijkstra's algorithm first loop.

Function Name:
compute_new_distances

For the current contact we compute the distances between this contact and all of its (unvisited) neighbors, then we compare the distance computed with the previous computed distance for the neighbor contact and assign the smaller one.

Date Written:
30/01/20
Returns
void
Parameters
[in]*sapThe current state of phase one
[in]current_timeThe internal time of Unibo-CGR
[in]*currentThe current node, we consider the toNode field of this contact to know who are the neighbours and at the end we add this node to the visited set
Notes:
  1. This is a modified Dijkstra's algorithm, so we have even the excluded set and not excluded set (ContactNote's suppressed field)
  2. A neighbor for the current contact is each contact that has as fromNode field equal to the toNode field of the current contact.
