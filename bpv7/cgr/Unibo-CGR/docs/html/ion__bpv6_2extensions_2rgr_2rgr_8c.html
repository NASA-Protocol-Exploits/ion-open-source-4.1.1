Bundle *  bundle

Function Name: rgr_offer
Purpose: This callback aims to ensure that the bundle contains
a RGR extension block keep track of the source routing. If the bundle already contains such a RR block (inserted by an upstream node) then the function simply returns 0. Otherwise the function creates an empty RR block to be populated during the ION execution.
Returns
int
Return values
0The RGR was successfully created, or not needed.
-1There was a system error.
Parameters
[in,out]blkThe block that might be added to the bundle.
[in]bundleThe bundle that would hold this new block.
Notes:
  1. Code was written with inspiration to the bpsec extension blocks developed by E. Birrane.
  2. All block memory is allocated using sdr_malloc.

20/11/18 L. Mazzuca Initial Implementation and documentation
