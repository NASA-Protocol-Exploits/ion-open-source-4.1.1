Decode (CBOR decoding) a buffer into a GeoRoute.

Function Name:
decode_rgr

This function expects to decode an open array of size 2 which contains an integer (the lenght of the text string) and a text string

Date Written:
11/04/20
Returns
int
Return values
1GeoRoute decoded correctly
0GeoRoute decoded correctly but the buffer contains other unparsed bytes
-1Decoding error
-2Arguments error
-3MTAKE error
Parameters
[in]bytesBufferedThe number of bytes contained in bufferToDecode
[in]*bufferToDecodeThe buffer that contains the GeoRoute encoded with CBOR encoding. The buffer should be encoded by encode_rgr function.
[out]geoRouteThe GeoRoute decoded.
[in]TODO_MTAKE_nodesBoolean: set to 1 if you want that this function allocate memory (with MTAKE) for geoRoute->nodes, otherwise set to 0.
Notes:
  1. You should call this function to decode a buffer encoded by encode_rgr.
  2. Remember to deallocate the geoRoute->nodes with MRELEASE if you choose to allocate it into this function.

Added by A. Stramiglio 26/11/19.

Function Name: rgr_read
Purpose: This function converts an ExtensionBlock into a GeoRoute Struct.
if the ExtensionBlock is empty nothing changes. The GeoRoute Struct passed must be initialized. The function allocates an rgrBlk in psm Memory with MTAKE.The caller needs to do the MRELEASE.
Returns
int
Return values
0The GeoRoute was successfully filled.
-1There was a system error.
-2Empty GeoRoute (rgrBlk)
-3Empty ExtensionBlock (blk)
Parameters
[in,out]blkThe ExtensionBlock read by the sdrMemory
[in,out]rgrBlkThe GeoRoute filled with ExtensionBlock values.
Notes:
  1. All block memory are allocated using MTAKE.
