◆ ADD_AND_REVISE_CONTACT

#define ADD_AND_REVISE_CONTACT

Boolean: set to 1 if you want to permits that the contact's xmit rate and confidence can be changed when you are trying to insert into the contact graph a contact that already exists.

Notes:
  1. You can find this macro helpful when you copy the contacts graph from an interface (so you just convert the contact) for a BP implementation.
  2. If you read the contacts graph from a file with an instruction (i.e. "add contact") maybe you prefer to disable this macro and instead add a "change contact" instruction.
  3. Just to avoid disambiguity: you can revise a contact only if you're adding a contact with the same {fromNode, toNode, fromTime}.

Returns
int
Return values
2Success case: Revised xmit rate (and MTVs)
1Success case: contact added to the contacts graph
0Arguments error case
-1Overlapped contact (don't change it)
-2MWITHDRAW error
Parameters
[in]fromNodeThe contact's sender node
[in]toNodeThe contact's receiver node
[in]fromTimeThe contact's start time

  2. Set copyMTV to 1 if you want to be compatible with the data gets for example from an interface.
