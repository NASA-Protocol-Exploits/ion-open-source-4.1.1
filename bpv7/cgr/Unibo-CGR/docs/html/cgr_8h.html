Implementation of the CGR, get the best routes list.

Function Name:
getBestRoutes
Date Written:
15/02/20
Returns
int
Return values
>= 0Success case: number of best routes found
-1There aren't routes to reach the destination.
-2MWITHDRAW error
-3Phase one error (phase one's arguments error)
-4Arguments error
-5Time is in the past
Parameters
[in]timeThe current time
[in]*bundleThe bundle that has to be forwarded
[in]excludedNeighborsThe excluded neighbors list, the nodes to which the bundle hasn't to be forwarded as "first hop"
[out]*bestRoutesIf result > 0: the list of best routes, NULL otherwise
Notes:
  1. If the contact plan changed this function discards all the routes computed by the previous calls.
  2. You must initialize the CGR before to call this function.
