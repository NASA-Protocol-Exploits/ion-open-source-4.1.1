Detailed Description

definitions supporting implementation of the CGR Route extension block. All functions defined here are requested and used by libbpP, except cgrr_attach. Other functions specific to the use of this extension block can be found in cgrr_util.

Function Name: cgrr_attach
Purpose: Compute and attach a CGRR block within the bundle.
Returns
int
Return values
-1Error.
>= 0CGRRouteBlock Attached
Parameters
[out]blkThe serialized CGRR extension block.
[out]cgrrBlkThe CGRRouteBlock to be serialized.
Notes:
  1. The ExtesionBlock blk MUST be pre-allocated and of the correct size to hold the CGRRouteBlock.
  2. The passed-in CGRRouteBlock MUST be pre-initialized with at least default values.
  3. The CGRRouteBlock MUST be pre-allocated and initialized with a size, and the object within the block MUST be allocated in the SDR.

int cgrr_offer ( ExtensionBlock *  blk,
Bundle *  bundle

Function Name: cgrr_offer
Purpose: This callback aims to ensure that the bundle contains
a CGRR extension block to keep track of the source routing. If the bundle already contains such a CGRR block (inserted by an upstream node) then the function simply returns 0. Otherwise the function creates an empty CGRR block to be populated during the CGR execution.
Returns
int
Return values
0The CGRR was successfully created, or not needed.
-1There was a system error.
Parameters
[in,out]blkThe block that might be added to the bundle.
[in]bundleThe bundle that would hold this new block.
Notes:
  1. This code is inspired by the bpsec extension blocks designed by E. Birrane.
  2. All block memory is allocated using sdr_malloc.
